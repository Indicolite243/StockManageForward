## 1. 账户信息模块

### API接口
```
GET /api/account-info/
```

### 请求参数
无

### 响应数据格式
```json
{
  "accounts": [
    {
      "account_id": "DEMO000001",
      "account_type": "STOCK",
      "total_asset": 4100000.00,
      "cash": 1250000.00,
      "frozen_cash": 75000.00,
      "market_value": 2850000.00,
      "positions": [
        {
          "account_id": "DEMO000001",
          "account_type": "STOCK",
          "stock_code": "600519.SH",
          "stock_name": "贵州茅台",
          "volume": 500,
          "can_use_volume": 500,
          "open_price": 1680.50,
          "market_value": 840250.00,
          "frozen_volume": 0,
          "on_road_volume": 0,
          "yesterday_volume": 500,
          "avg_price": 1620.00
        }
      ]
    }
  ]
}
```

### 字段说明

#### 账户级别字段
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account_id | string | 是 | 账户ID |
| account_type | string | 是 | 账户类型（STOCK等） |
| total_asset | float | 是 | 总资产（元） |
| cash | float | 是 | 可用金额（元） |
| frozen_cash | float | 是 | 冻结金额（元） |
| market_value | float | 是 | 持仓市值（元） |

#### 持仓级别字段
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| stock_code | string | 是 | 股票代码 |
| stock_name | string | 是 | 股票名称 |
| volume | int | 是 | 持仓数量 |
| can_use_volume | int | 是 | 可用数量 |
| open_price | float | 是 | 当前价格 |
| market_value | float | 是 | 股票市值（元） |
| frozen_volume | int | 是 | 冻结数量 |
| on_road_volume | int | 是 | 在途数量 |
| yesterday_volume | int | 是 | 昨日持仓 |
| avg_price | float | 是 | 成本价 |

### 前端使用场景
- 资产展示模块：显示账户总资产、可用金额、持仓市值等
- 持仓详情：显示前10只持仓股票的详细信息

## 2. 资产对比模块

### API接口
```
GET /api/asset_comparison/?account_id=DEMO000001
```

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account_id | string | 是 | 账户ID |

### 响应数据格式
```json
{
  "asset_data": [
    {
      "stock_code": "600519.SH",
      "stock_name": "贵州茅台",
      "market_value": 840250.00,
      "asset_ratio": 29.48,
      "percentage": 29.48,
      "daily_return": 15.08,
      "profit_loss_rate": 15.08
    }
  ],
  "positions": [
    {
      "stock_code": "600519.SH",
      "stock_name": "贵州茅台",
      "market_value": 840250.00,
      "asset_ratio": 29.48,
      "daily_return": 15.08
    }
  ]
}
```

### 字段说明
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| stock_code | string | 是 | 股票代码 |
| stock_name | string | 是 | 股票名称 |
| market_value | float | 是 | 市值（元） |
| asset_ratio | float | 是 | 资产占比（%） |
| percentage | float | 否 | 资产占比（%，与asset_ratio二选一） |
| daily_return | float | 是 | 日收益率（%） |
| profit_loss_rate | float | 否 | 盈亏率（%，与daily_return二选一） |

### 前端使用场景
- 资产对比图表：饼图、柱状图展示
- 资产对比表格：列表展示股票资产信息
- 业绩归因模块：用于计算市值分布

### 注意事项
- 前端兼容两种字段名：`asset_data` 和 `positions`
- 前端兼容两种占比字段：`asset_ratio` 和 `percentage`
- 前端兼容两种收益率字段：`daily_return` 和 `profit_loss_rate`

## 3. 时间对比模块

### API接口
```
GET /api/timecomparison/yearly_comparison/?account_id=DEMO000001
```

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account_id | string | 是 | 账户ID |

### 响应数据格式
```json
{
  "yearly_data": [
    {
      "year": "2023",
      "totalAssets": 3200000.00,
      "returnRate": 12.50,
      "investmentRate": 8.30
    },
    {
      "year": "2024",
      "totalAssets": 3680000.00,
      "returnRate": 15.00,
      "investmentRate": 9.50
    },
    {
      "year": "2025",
      "totalAssets": 4100000.00,
      "returnRate": 11.41,
      "investmentRate": 7.80
    }
  ]
}
```

### 字段说明
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| year | string | 是 | 年份（YYYY格式） |
| totalAssets | float | 是 | 总资产（元） |
| returnRate | float | 是 | 回报率（%） |
| investmentRate | float | 是 | 投资占比（%） |

### 前端使用场景
- 年度对比图表：折线图、柱状图展示
- 年度对比表格：列表展示年度数据

## 5. 风险阈值模块

### API接口
```
GET /api/risk-threshold/assessment/?account_id=DEMO000001&days=30
```

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account_id | string | 是 | 账户ID |
| days | int | 否 | 统计天数（默认30天） |

### 响应数据格式
```json
{
  "account_id": "DEMO000001",
  "max_principal_loss": {
    "max_loss_rate": 1.22,
    "status": "正常"
  },
  "volatility": {
    "annual_volatility": 23.02,
    "status": "正常"
  },
  "max_drawdown": {
    "max_drawdown": 5.25,
    "status": "正常"
  },
  "var": {
    "var_rate": 1.50,
    "status": "正常"
  }
}
```

### 字段说明

#### 最大本金损失
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| max_loss_rate | float | 是 | 最大损失率（%） |
| status | string | 是 | 状态（"正常"、"警告"、"危险"） |

#### 波动率
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| annual_volatility | float | 是 | 年化波动率（%） |
| status | string | 是 | 状态（"正常"、"警告"、"危险"） |

#### 最大回撤
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| max_drawdown | float | 是 | 最大回撤（%） |
| status | string | 是 | 状态（"正常"、"警告"、"危险"） |

#### VaR值
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| var_rate | float | 是 | VaR值（%） |
| status | string | 是 | 状态（"正常"、"警告"、"危险"） |

### 状态映射
前端会将后端的状态字符串映射为：
- "正常" → "normal"
- "警告" → "warning"
- "危险" → "danger"

### 前端使用场景
- 风险阈值表格：展示各项风险指标
- 风险总览：统计正常/预警/风险数量

## 6. 业绩归因模块

### API接口
```
GET /api/asset_comparison/?account_id=DEMO000001
```

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account_id | string | 是 | 账户ID |

### 响应数据格式
与[资产对比模块](#2-资产对比模块)相同。

前端会从 `asset_data` 或 `positions` 中提取数据，转换为：
```json
[
  {
    "name": "贵州茅台",
    "marketCap": 0.84,
    "category": "small"
  }
]
```

### 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| name | string | 股票名称 |
| marketCap | float | 市值（亿元） |
| category | string | 市值分类（"small"：<50亿，"medium"：50-500亿，"large"：>=500亿） |

### 前端使用场景
- 业绩归因图表：展示市值分布

## 📝 通用规范

### 1. 数据格式
- 所有金额单位：**元**（不是万元或亿元，前端会根据需要转换）
- 所有百分比：**数字类型**（如 12.5 表示 12.5%），某些接口可能返回字符串（如 "12.5%"）
- 日期格式：**YYYY-MM-DD** 或 **YYYY-MM-DD HH:mm:ss**

### 2. 错误处理
- 如果API请求失败，前端会回退到模拟数据
- 后端应返回标准的HTTP状态码（200成功，400客户端错误，500服务器错误）
- 错误响应格式：
```json
{
  "error": "错误描述",
  "detail": "详细错误信息"
}
```

### 3. 空值处理
- 如果某个字段为空，应返回 `null` 或 `0`（根据字段类型）
- 数组字段如果为空，应返回空数组 `[]`

### 4. 兼容性
- 前端对某些字段名做了兼容处理，支持多种字段名（如 `asset_data` 和 `positions`）
- 建议后端统一使用标准字段名

### 5. 数据精度
- 金额字段：保留2位小数
- 百分比字段：保留2位小数
- 价格字段：保留4位小数

## 📌 重要提醒

1. **所有API接口的baseURL为**：`http://localhost:8000`
2. **前端会在请求失败时回退到模拟数据**，所以后端接口必须稳定可用
3. **CORS配置**：后端需要配置允许前端域名（`http://localhost:5173`）的跨域请求
4. **数据实时性**：前端会定期刷新数据，建议后端提供实时或准实时的数据
5. **字段命名**：建议使用下划线命名（snake_case），如 `stock_code`、`market_value`
